# 目前已经实现的功能

更新日期：2026-07-13

## Agent Runtime

- 已实现 OpenAI-compatible 与 Anthropic-compatible 双协议流式 Agent Loop，覆盖模型请求、Tool Call 解析、工具结果回写与连续推理，核心代码位于 `agents/agent.py`。
- 已实现交互式 REPL 与一次性任务模式，支持 `--resume`、`--plan`、`--max-cost`、`--max-turns` 等启动参数，入口位于 `agents/main.py`。
- 已实现会话自动保存、最近会话恢复和多层上下文压缩，相关代码位于 `agents/session.py` 与 `agents/agent.py`。

## 工具与权限

- 已实现文件读取、写入、精确编辑、目录浏览、代码搜索和 Shell 命令等内置工具，相关代码位于 `agents/tools.py`。
- 已实现默认确认、自动接受编辑、只读 Plan Mode、自动拒绝和跳过确认等权限模式。
- 已实现延迟工具发现以及只读工具的并发执行能力。

## Memory

- 已实现按项目隔离的文件型长期记忆、`MEMORY.md` 索引、相关记忆筛选和异步注入，相关代码位于 `agents/memory.py`。
- 已修复 Windows 下同一项目因路径大小写不同而生成多个 Memory 项目目录的问题：项目路径在计算哈希前会经过 `Path.resolve()` 和 `os.path.normcase()` 规范化。
- 已增加 Windows 路径大小写回归测试 `tests/test_memory.py`。

## Skills、MCP 与子 Agent

- 已实现项目级和用户级 `SKILL.md` 的发现、检索、调用、反馈与版本演化。
- 已实现从用户反馈中抽取在线 Skill 候选，并支持 add、merge、discard 决策与来源审计。
- 已实现 stdio JSON-RPC MCP Client，可加载外部 MCP Server 工具。
- 已实现 `explore`、`plan`、`general` 和自定义子 Agent。

## 最近验证

- `python -m unittest discover -s tests -v`：通过（1 个测试），确认 Windows 路径大小写映射到同一项目哈希。
- `python -m compileall -q agents tests`：通过，确认 Agent 与测试模块可以完成 Python 字节码编译。
