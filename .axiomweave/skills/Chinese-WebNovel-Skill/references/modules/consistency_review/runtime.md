# 审查模块运行规则

## 一、这是每章完稿后的强制流程

默认顺序：

1. 写完一章
2. 不急着交付
3. 先调用本模块做一致性审查
4. 审查不过则回对应模块修
5. 审查通过再输出正文

## 二、逐章必查的六项

### 1. 剧情逻辑一致性

检查问题：

- 这一章最关键的事件，前提够不够
- 最关键的决定，触发够不够
- 事件发生后，局面有没有实质变化

不过关时回：

- `plot_logic/runtime.md`

### 2. 人物目标一致性

检查问题：

- 主角本章目标是否清楚
- 过程中是否无故偏移

不过关时回：

- `character_consistency/runtime.md`

### 3. 情绪与关系一致性

检查问题：

- 情绪是否有触发和过渡
- 关系状态是否和上一场接得上

不过关时回：

- `character_consistency/runtime.md`
- 必要时联动 `transition/runtime.md`

### 4. 身体与信息状态一致性

检查问题：

- 伤势、疲劳、怀孕、药效、秘密、误会、已知线索有没有丢

不过关时回：

- `character_consistency/runtime.md`
- `plot_logic/runtime.md`

### 5. 场景与转场一致性

检查问题：

- 读者会不会迷路
- 时间、空间、视角是否清楚
- 上一场余力有没有被带过去

不过关时回：

- `transition/runtime.md`

### 6. 章末承接一致性

检查问题：

- 章末是否收在变化上
- 下一章第一拍是否能接住

不过关时回：

- `chapter_ending/runtime.md`
- `transition/runtime.md`

## 三、最小审查输出模板

完稿后默认先做一轮内部检查：

```text
1. 剧情逻辑：稳 / 不稳，问题在……
2. 目标一致：稳 / 不稳，问题在……
3. 情绪关系：稳 / 不稳，问题在……
4. 身体信息：稳 / 不稳，问题在……
5. 场景转场：稳 / 不稳，问题在……
6. 章末承接：稳 / 不稳，问题在……
```

任意两项不稳，就不要直接交稿。

## 四、专项硬规则

- 不要只查病句，不查结构。
- 不要只查逻辑，不查人物。
- 不要只查本章，不看下一章入口。
- 不要把审查当附加步骤，它是交付步骤的一部分。
