# 分卷章纲模块运行规则

## 一、什么时候必须调用

出现这些信号时，优先调用本模块：

- 用户要分卷
- 用户要章纲
- 用户要规划前 10-20 章
- 用户担心中段发散
- 已经有设定和开头，但没有稳定连载结构

## 二、先判断卡在哪一层

### 第 1 层：黄金三章层

典型表现：

- 前三章很慢
- 卖点没有递进
- 第三章还没立方向

优先补：

- 第一章异常
- 第二章卖点展开
- 第三章追更理由

### 第 2 层：第一卷层

典型表现：

- 知道开头，后面不知道怎么跑

优先补：

- 卷目标
- 卷冲突
- 卷高潮
- 卷末变化

### 第 3 层：章纲层

典型表现：

- 每章只有事件，没有拉力

优先补：

- 本章目标
- 本章冲突
- 新信息
- 章末拉力

### 第 4 层：中段扩展层

典型表现：

- 支线乱长
- 设定膨胀
- 更新感变弱

优先补：

- 卷目标约束
- 章节删并
- 阶段兑现点

## 五、任务类型处理

### 1. 如果用户要你诊断大纲

先输出：

- 卡在黄金三章、第一卷、章纲还是中段扩展
- 最容易散在哪里

### 2. 如果用户要你做分卷

先做：

- 定故事引擎
- 定卷目标
- 定卷末变化

### 3. 如果用户要你拆章纲

默认顺序：

1. 先有卷目标
2. 再拆前 10-20 章
3. 每章都写目标、冲突、新信息、章末拉力

## 六、专项硬规则

- 不要只有卷名，没有卷目标。
- 不要章纲只记“发生了什么”。
- 不要前三章只讲背景。
- 不要把支线放进来却不服务卷目标。
