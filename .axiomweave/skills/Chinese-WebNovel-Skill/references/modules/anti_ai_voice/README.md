# 去 AI 味模块

这是一个专门处理中文网文表达去模板化、去空泛化、去同质化的专项模块。

它不把“去 AI 味”理解成最后一轮修辞润色，而是把它放回写作流程里，处理这些更根本的问题：

- 句子只会总结，不会落动作
- 情绪全靠解释，不会自己长出来
- 人物说话一个腔调，身份感不见了
- 气氛词、形容词、网文套词堆得很满，但场面没有真正发生
- 文字表面顺，但没有压强、没有质感、没有具体物

## 什么时候调用

当任务明确出现这些信号时，优先调用本模块：

- 用户明确说“AI 味重”“太像模型写的”“太模板化”
- 文本里高频出现空泛总结、抽象情绪、套话氛围词
- 每个人说话都很工整、很会解释、没有身份差
- 想修的不是剧情大纲，而是句面层、场面层、人物声音层的同质化
- 已经有剧情骨架，但正文一写就发虚、发空、发假

## 建议阅读顺序

1. 先读 [tutorial.md](tutorial.md)，统一“AI 味”到底指什么。
2. 再读 [runtime.md](runtime.md)，按层级判断问题卡在哪。
3. 去 [good_examples.md](good_examples.md) 挑 `2-4` 个近似正例。
4. 去 [bad_examples.md](bad_examples.md) 挑 `1-2` 个近似反例。
5. 先写一版“替换计划”，再动正文。

## 文件说明

- [tutorial.md](tutorial.md)
  把“去 AI 味”从口号拆成可执行的判断框架。
- [runtime.md](runtime.md)
  给模型的实际运行顺序。
- [good_examples.md](good_examples.md)
  汇总可直接迁移的具体化、动作化、人物声音化正例。
- [bad_examples.md](bad_examples.md)
  汇总最常见的空泛、解释腔、套话腔反例。
- [source_index.md](source_index.md)
  标明本模块主要依据哪些现有资料重组。

## 说明

- 这个模块不替代剧情、转场、对白模块，而是给这些模块最后落正文时补一层“句面真实度”。
- 如果文本首先卡在因果、人物动机、转场承接，那要先修更底层问题，再回来去 AI 味。
