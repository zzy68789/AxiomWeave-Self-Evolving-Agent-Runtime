# 开头模块

这是一个专门处理网文开头问题的专项模块。

它不解决“全文怎么规划”，而是专门解决：

- 开头起得慢
- 有句子，没有抓手
- 有热闹，没有卖点
- 有设定，没有主角
- 有主角，没有异常局面
- 黄金三章开了三章还没把读者留下来

## 什么时候调用

当任务明确落在这些问题上时，优先调用本模块：

- 用户要写开头、改开头、重开篇
- 用户说“开头没劲”“前几百字抓不住人”
- 黄金三章推进无力
- 读者会问“所以这本书到底卖什么”

## 建议阅读顺序

1. 先读 [tutorial.md](tutorial.md)，统一开头的判断框架。
2. 再读 [runtime.md](runtime.md)，按层级诊断问题。
3. 去 [good_examples.md](good_examples.md) 选 `2-4` 个正例。
4. 去 [bad_examples.md](bad_examples.md) 选 `1-2` 个反例避坑。
5. 先写开头计划，再写正文。

## 文件说明

- [tutorial.md](tutorial.md)
  把“开头”从题材卖点、异常局面、信息外挂、关系炸点几个角度重组为一套可执行方法。
- [runtime.md](runtime.md)
  给模型的实际调用顺序。
- [good_examples.md](good_examples.md)
  以现有语料中的开头钩子摘录为主，重点看“第一拍为什么能成交”。
- [bad_examples.md](bad_examples.md)
  以高频失败开头为主，重点看“为什么读者会滑走”。
- [source_index.md](source_index.md)
  标明本模块主要借用了哪些主 skill 内容和哪些本地语料摘录。
