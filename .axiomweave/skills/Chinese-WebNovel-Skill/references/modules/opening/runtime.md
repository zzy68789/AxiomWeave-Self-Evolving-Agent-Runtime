# 开头模块运行规则

## 一、什么时候必须调用

出现这些信号时，优先调用本模块：

- 用户要开篇、重写开头、优化黄金三章
- 文本前 500 字没有事件抓手
- 前三章还没明确卖点和主角目标
- 开头先讲背景、设定、规则，再讲故事
- 看起来热闹，但读者读完不知道后续要追什么

## 二、先判断问题卡在哪一层

### 第 1 层：新手型慢热

典型表现：

- 从天气、起床、环境、回忆起手
- 三五段过去还没出现异常
- 第一场戏没有压力

优先补：

- 抓手
- 异常
- 当前代价

### 第 2 层：有热闹，没有卖点

典型表现：

- 一上来很吵，但不知道这本书到底卖什么
- 事件发生了，但没有延展引擎

优先补：

- 题材消费点
- 故事引擎
- 第三章前的追更问题

### 第 3 层：有卖点词，没有局面

典型表现：

- 写了“重生”“真假千金”“系统”“弹幕”
- 但这些设定没立刻改变主角处境

优先补：

- 卖点落地
- 信息改轨
- 关系或利益变化

### 第 4 层：黄金三章递进失效

典型表现：

- 第一章立了局，第二章原地打转
- 第三章还没把方向立稳

优先补：

- 每章新增变化
- 短期目标
- 章末拉力

## 四、动笔前先写开头计划

正文前先明确：

- 第一刀切在什么异常上：
- 主角现在就要承受什么代价：
- 第一拍之后读者最想知道什么：
- 开头前三段里，卖点在哪一句或哪一刀露头：
- 第一章末要把局面推到哪里：

## 五、任务类型处理

### 1. 如果用户要你诊断开头

先输出：

- 现在卡在哪一层
- 现在缺的是抓手、卖点、局面还是递进
- 最该补的是哪一刀

### 2. 如果用户要你重写开头

先做：

- 保留原题材卖点
- 选一类更有力的起手方式
- 明确前三段分别承担什么

### 3. 如果用户要你从简介直接起开头

默认顺序：

1. 先抽题材卖点
2. 再选起手类型
3. 再写前三段计划
4. 最后写正文

## 六、专项硬规则

- 不要从天气和日常流程起手。
- 不要先讲世界观再讲人。
- 不要只抛热闹，不交代卖点。
- 不要把外挂当贴纸。
- 不要让前三章像同一场戏的重复说法。
