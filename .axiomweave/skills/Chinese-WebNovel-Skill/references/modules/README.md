# 网文专项模块索引

这些模块不是泛写作教程，而是针对某个容易反复出问题的小说环节，提供一套可调用的诊断、选材、规划和落笔流程。

它们的目标不是把问题讲得更抽象，而是让模型在碰到局部硬伤时，不靠空泛建议和句面润色硬凑过去。

## 什么时候优先调用专项模块

当任务已经明显收束到某一个写作环节时，优先走专项模块，而不是只做泛检索或泛建议。

常见信号：

- 用户明确说的是某个局部问题，比如转场、剧情逻辑、人物一致性、章末承接
- 当前文本的问题不在句子漂不漂亮，而在结构桥梁、因果、状态延续、信息组织
- 需要先匹配类似素材，再决定怎么写
- 只靠润色会掩盖问题，不会真正修掉问题

## 专项模块的通用调用顺序

1. 先判断这是不是专项问题，不要把所有问题都当成文风问题。
2. 再选对应模块，先读该模块的 `README.md`。
3. 再按模块自己的推荐顺序进入 `tutorial.md`、`runtime.md` 和例库，不要用总索引去覆盖模块自己的入口顺序。
4. 按模块自己的分层和分类法判断故障点，不要跳过诊断直接套技巧。
5. 在模块例库里匹配 `2-4` 个正例和 `1-2` 个反例，只借结构，不借句面。
6. 先写一版局部计划：
   上一拍留下什么力；
   下一拍必须交代什么变化；
   哪些人物状态、关系状态、信息状态不能丢。
7. 最后再输出诊断、改写方案或正文。

## 使用提醒

- 模块不是拿来替代正文的，而是拿来决定正文该怎么写。
- 模块不是越多越好，只有当问题已经明确集中在某个环节时才调。
- 如果一个问题同时跨多个模块，先修更底层、更硬的那一层，再修更表层的问题。
