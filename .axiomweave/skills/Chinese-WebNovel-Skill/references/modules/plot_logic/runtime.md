# 剧情逻辑模块运行规则

## 一、什么时候必须调用

出现这些信号时，优先调用本模块：

- 用户明确说“逻辑不对”“剧情不通”
- 人物做决定前缺少必要触发
- 重要事件像巧合硬推
- 反转和误会只为效果服务，没有前提
- 章节推进像作者搬运，不像局面自然生长

## 二、先判断卡在哪一层

### 第 1 层：动机层

典型表现：

- 人物想做某事，但理由薄
- 人物明明没必要冒险，却硬冒险

优先补：

- 欲望
- 恐惧
- 现实压力

### 第 2 层：触发层

典型表现：

- 事情会发生，但“为什么现在发生”不清

优先补：

- 导火索
- 倒计时
- 外部逼迫

### 第 3 层：决策层

典型表现：

- 人物做法和既有人设、处境不匹配

优先补：

- 备选方案为什么行不通
- 当前选择为什么最符合他

### 第 4 层：后果层

典型表现：

- 事件发生了，但没有实质变化

优先补：

- 局面变化
- 关系变化
- 资源变化
- 风险变化

### 第 5 层：兑现层

典型表现：

- 前面埋了很多，后面没用
- 后面想收一个大点，前面没种
- 反转、误会、掉马、打脸只靠最后一翻

优先补：

- 回收点
- 兑现方式

## 五、任务类型处理

### 1. 如果用户要你诊断逻辑

先输出：

- 断在动机、触发、决策、后果还是兑现
- 具体哪一环缺了

### 2. 如果用户要你修逻辑

先做：

- 不改卖点，先补因果链
- 尽量补最少但最关键的一步

### 3. 如果用户要你直接创作

默认先写因果链，再写正文。

## 六、专项硬规则

- 不要用巧合代替触发。
- 不要用作者意图代替人物动机。
- 不要让大事件没有后果。
- 不要让反转只靠最后一句强翻。
