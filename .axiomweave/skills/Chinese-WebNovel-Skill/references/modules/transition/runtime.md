# 转场模块运行规则

这份文件不是教程，而是给模型的执行顺序。

它现在按 `raw` 资料里的分层思路重写：先判断问题在哪一层，再决定调哪类方法。

## 一、什么时候必须调用转场模块

当任务出现这些信号时，优先调用本模块，而不是直接改句子：

- 用户明确提到：转场、过渡、场景切换、承接、接不上
- 文本里高频出现：`第二天 / 后来 / 与此同时 / 回到家 / 几天后 / 另一边`
- 上一场和下一场的时间、地点、视角或情绪明显变了
- 人物上一场和下一场像两个人
- 章末钩子蓄了力，下一场没接住
- 回忆、梦境、插叙进出边界不清

## 二、先判断这是哪一层问题

不要一上来就说“这段转场不顺”。

先判断问题属于哪一层：

### 第 1 层：新手型硬切问题

典型表现：

- 只有“第二天”“回到家”
- 完全靠空行硬断
- 场景换了，但没有桥

优先考虑：

- 标志性短语
- 排版转场
- 简单时间桥
- 简单空间桥

### 第 2 层：分类型问题

典型表现：

- 时间跳切不顺
- 空间切换空空的
- 情绪承接断裂
- 视角切换像插播

优先判断它属于：

- 时间转场
- 空间转场
- 情绪转场
- 视角转场
- 话题/信息转场

### 第 3 层：镜头与关联问题

典型表现：

- 表面上有时间词，但还是硬
- 两场戏之间没有共同触点
- 作者切得出来，读者接不过去

优先考虑：

- 借物件
- 借细节
- 借环境空镜
- 借共同事件
- 摄像机接力
- 蒙太奇

### 第 4 层：网文节奏问题

典型表现：

- 转得顺，但太慢
- 转得快，但因果断
- 情绪很足，但拖
- 章节钩子和正文节奏脱节

这时要回到网文要求：

- 快
- 准
- 不迷路
- 第一拍就有东西

## 三、再匹配方法，不要反过来

正确顺序是：

1. 先判层级
2. 再判类型
3. 再选方法

例如：

- 如果是“时间词太空”，优先补后果，不要直接上蒙太奇
- 如果是“群像插线乱”，先找共同事件，不要只加“与此同时”
- 如果是“回忆太硬”，先找触发物和打断点

## 四、匹配素材的最低配置

每次至少匹配：

- 2-4 个正例
- 1-2 个反例

匹配时优先看：

- 这次切的是时间、空间、视角还是情绪
- 题材是否接近
- 场景强度是否接近
- 这是快切还是柔切

## 四点五、从哪一组例库里挑材料

不要每次都从整份例库里乱翻。

优先按下面这张映射表选：

### 1. 如果问题是“时间词太空 / 第二天太硬 / 后来像摘要”

先看：

- 正例：`good_examples.md` 的 A 组
- 反例：`bad_examples.md` 的 A 组、B 组

重点学：

- 时间词后面怎么立刻给后果
- 长跨度怎么只留阶段节点

### 2. 如果问题是“到了某地但没戏 / 场景换了却空”

先看：

- 正例：`good_examples.md` 的 B 组
- 反例：`bad_examples.md` 的 A 组、B 组

重点学：

- 进场第一眼怎么见人、见异常、见新信息
- 空间推进怎么顺手带出事件

### 3. 如果问题是“需要一个触发点 / 消息来了却没改剧情”

先看：

- 正例：`good_examples.md` 的 C 组
- 反例：`bad_examples.md` 的 A 组、B 组

重点学：

- 电话、弹幕、短信、突发动作怎么做改轨器
- 新信息怎么逼人物改判断、改行动

### 4. 如果问题是“想转得更巧 / 想靠物件、细节、感官来切”

先看：

- 正例：`good_examples.md` 的 D 组
- 反例：`bad_examples.md` 的 D 组、E 组

重点学：

- 借物件
- 借细节变化
- 借感官突变
- 借环境或不变之物反衬时间变化

### 5. 如果问题是“章末和下一场接不上”

先看：

- 正例：`good_examples.md` 的 E 组
- 反例：`bad_examples.md` 的 B 组

重点学：

- 上一章留下的力怎么带到下一场
- 下一个场景第一拍怎么兑现、反打或升级

### 6. 如果问题是“人物崩了 / 情绪断了 / 身体状态忘了”

先看：

- 正例：`good_examples.md` 的 A 组、B 组、C 组里最接近的例子
- 反例：`bad_examples.md` 的 C 组

重点学：

- 状态变化前必须补什么触发物
- 关系、伤势、目标怎么跨场景延续

### 7. 如果问题是“视角切换乱 / 回忆梦境边界不清”

先看：

- 正例：`good_examples.md` 的 C 组、D 组
- 反例：`bad_examples.md` 的 D 组

重点学：

- 共同事件
- 触发物
- 打断点
- 回到现实的边界提示

### 选材补充规则

- 先从目标组挑 `2` 个。
- 再从相邻组补 `1-2` 个。
- 如果一个问题同时跨两组，就先解决更靠前的硬伤，再处理更高阶的问题。

例如：

- “第二天太硬”先补 A 组，不要直接去抄 D 组的物件转场。
- “视角乱”先补共同事件，不要先加情绪描写。
- “章末没接住”先看 E 组，不要先润色句子。

## 五、动笔前先写一版“桥梁计划”

正文之前，先在脑中明确这 6 句：

- 上一场留下的未尽之力是什么：
- 下一场第一拍必须交代什么变化：
- 这次是借时间、借空间、借情绪，还是借物件：
- 这次最自然的桥梁动作或共同触点是什么：
- 哪些人物状态绝对不能丢：
- 这次切换要让读者明显感觉到，还是尽量无痕：

如果这 6 句说不清，先不要写正文。

## 六、不同任务的默认处理方式

### 1. 如果用户要你诊断问题

先输出：

- 它卡在哪一层
- 它属于哪一类转场
- 具体断在哪里
- 最该补的是桥梁动作、后果、触发物，还是共同事件

### 2. 如果用户要你改写现成文本

先做：

- 标出上一场的余力
- 标出下一场的目标
- 选 1 个桥梁触点
- 再决定是快切、柔切还是借物切

再重写，不要直接塞一个“第二天”。

### 3. 如果用户要你直接创作新正文

先做：

- 判层级
- 判类型
- 匹配正反例
- 摘出可迁移结构
- 确定这次镜头怎么切

再进正文。

## 七、专项硬规则

- 不要把“第二天”“后来”“与此同时”当完整解决方案。
- 不要让转场前后人物状态无故跳变。
- 不要用作者总结替代桥梁动作。
- 不要把环境描写写成长篇散文后才落剧情。
- 不要为了“无痕”把因果省掉。
- 不要只顾“高级感”而让读者迷路。
- 不要照抄例子句面，只能借结构。

## 八、写完后的自查问题

- 读者进入下一场会不会迷路
- 这一场第一句有没有新变化
- 上一场的力有没有被带过来
- 这次切换有没有可见的桥梁
- 人物情绪、关系、伤势、目标有没有断
- 如果删掉这次转场，中间会不会少一整步逻辑

只要其中任意两项答“不稳”，就回去重写。
