# 前置规划教程

## 一、先统一一个定义

前置规划不是把设定写得更大，而是：

`把一句简介压成能连载、能拆章、能继续往下写的最小骨架。`

如果这一层不稳，后面的开头、章纲、正文都会越来越虚。

## 二、前置规划要先回答的六个问题

### 1. 题材和消费点是什么

不是只说“古言”“都市”“仙侠”，还要说清：

- 读者主要来吃什么
- 这条题材最强的爽点、甜点、刀点在哪
- 这本书到底更像事业反打、关系拉扯、求生升级，还是悬念驱动

### 2. 卖点和异常局面亮不亮

一句话里最好能看见：

- 谁
- 在什么异常局面里
- 因为什么必须做什么
- 最抓人的点是什么

卖点不是世界观总论，而是最先能成交的那一下。

### 3. 主角眼前的麻烦成不成立

主角不能只是“很特别”“背景复杂”。

还要让读者立刻知道：

- 他眼前最紧迫的问题是什么
- 如果不动，会失去什么
- 第一章到第三章最先会撞上什么麻烦

### 4. 故事引擎能不能持续推进

好点子不只要能开头，还要能往下跑。

要先看：

- 接下来 10-20 章靠什么持续推进
- 是升级、调查、身份反打、关系拉扯，还是倒计时逼迫
- 每推进一步，局面会改写什么

### 5. 目标长度合不合理

不是所有点子都适合长篇。

要判断：

- 这个点子适合短篇、中篇、长篇还是连载
- 它有没有足够多的阶段性变化
- 它能不能自然拆出卷目标和卷末变化

### 6. 能不能压成一组最小产物

前置规划至少要落成这些东西：

- 一句话 hook
- 一段 premise
- 主角当前目标
- 主要阻力源
- 故事引擎说明
- 第一卷最小目标

如果这些东西压不出来，正文一般也写不稳。

## 三、最常见的前置规划问题

### 1. 设定很大，事件很弱

世界观看起来热闹，但读者不知道第一章到底要看什么。

### 2. 卖点很抽象

只会说“成长”“救赎”“命运”，没有具体的异常局面和可见冲突。

### 3. 主角很特别，但没有被逼着动

设定是存在的，麻烦却不落到眼前，所以故事开不起来。

### 4. hook 像世界观简介

信息很多，但没有把最能成交的点推到前面。

### 5. premise 像背景资料

全在解释历史、规则、阵营，没有主角当前处境和眼前麻烦。

### 6. 第一卷方向不明

知道这本书大概要写什么，却不知道卷一先赢什么、先失去什么、先变成什么。

## 四、前置规划时，不要先铺正文

先做：

- 题材判断
- 卖点压缩
- hook
- premise
- 故事引擎
- 第一卷方向

再做：

- 开头
- 黄金三章
- 章纲
- 单章正文

## 五、写之前的最小骨架卡

动笔前至少写清：

- 这是一本什么题材的网文：
- 读者主要来吃什么：
- 主角眼前最紧迫的问题是什么：
- 接下来 10-20 章靠什么持续推进：
- 更适合什么长度：
- 第一卷最小目标和卷末变化是什么：

如果这六句写不清，不要急着开正文。
