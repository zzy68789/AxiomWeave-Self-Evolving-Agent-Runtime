# 前置规划模块

这是一个专门处理网文前置规划问题的专项模块。

它主要解决：

- 点子能不能写成网文不清
- 题材和消费点不清
- 卖点和异常局面不亮
- hook / premise 站不住
- 故事引擎跑不动
- 目标长度和第一卷方向不清

## 什么时候调用

当任务明确落在这些问题上时，优先调用本模块：

- 用户要你判断一个点子值不值得写
- 用户要你做选材、构思、hook、premise、故事引擎或长度判断
- 简介看上去有设定，但没有立刻能开的事件
- 前 10-20 章怎么跑还不清
- 还没开写正文，先要把骨架搭稳

## 建议阅读顺序

1. 先读 [tutorial.md](tutorial.md)，统一判断框架。
2. 再读 [runtime.md](runtime.md)，按层级定位卡点。
3. 去 [good_examples.md](good_examples.md) 选近似结构。
4. 去 [bad_examples.md](bad_examples.md) 找高频空转写法。
5. 先写最小骨架，再转去 `opening`、`volume_outline` 等下游模块。
